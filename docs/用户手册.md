# 第一部分：开发者启动指南

## 1. 项目概述

**一句话定位**：面向本科毕设生的问卷研究预演工具，提前模拟数据方向及趋势，避免问卷"发出去石沉大海 / 收回来信效度不达标 / 相关性不显著"的白做工。

**合规底线**：定位为研究预演工具（A 路线），非数据生成工具（B 路线，学术造假帮凶）；全程 simulated 水印 + 免责声明；拒绝"题目不改强行造达标数据"的诉求。

### 1.1 系统架构

```
┌─────────────────────┐     同源 fetch      ┌──────────────────────┐    HTTP + dev-token    ┌─────────────────────┐
│  前端 Next.js 14    │  ───────────────▶  │  BFF (Next API Routes)│  ───────────────────▶  │  后端 FastAPI :8000 │
│  (localhost:3000)   │   /api/*            │  app/api/*/route.ts   │  /api/v1/*             │  SQLite + WAL      │
│  TanStack Query     │                     │  转发 + 格式转换       │                        │  DeepSeek LLM      │
└─────────────────────┘                     └──────────────────────┘                        └─────────────────────┘
```

- **前端**：Next.js 14（App Router）+ TypeScript strict + Tailwind + shadcn/ui + TanStack Query + Zustand
- **BFF**：Next.js API Routes，转发请求到后端并做格式转换，前端同源调用
- **后端**：Python FastAPI + SQLAlchemy 2.0 async + Alembic
- **数据库**：开发环境 SQLite（WAL 模式），生产环境 PostgreSQL
- **LLM**：DeepSeek 双模型（V3 做理解/推断/解析，R1 做诊断推理）
- **统计**：scipy + statsmodels + factor_analyzer + pingouin
- **导出**：python-docx（Word）+ openpyxl（Excel）

## 6. 数据库说明

### 6.1 SQLite 开发约定

- **WAL 模式**：已在 `database.py` 通过事件监听器配置 `PRAGMA journal_mode=WAL` + `PRAGMA busy_timeout=5000`，支持脚本直连与服务并发读写
- **WAL 读快照**：一个连接的读快照在事务期间固定，外部进程写入后需重新连接才能看到新数据（验证脚本查 DB 前需 `conn.close()` 重连）
- **并发**：避免脚本直连与服务同时长时间占用连接

### 6.2 UUID 存储格式

模型使用 `sqlalchemy.Uuid` 类型，在 SQLite 上以 `.hex`（32 位无连字符）存储。注意事项：
- API 流程无碍（pydantic 自动转 UUID→.hex）
- **外部脚本直连 DB 插入 UUID 时必须用 `.hex` 格式**（如 `uuid.uuid4().hex`），不能用 `str(uuid)`（带连字符），否则查询不匹配
- 历史脏数据：早期用带连字符格式插入的行需 UPDATE 为 `.hex` 格式

### 6.3 迁移命令

```bash
# 应用迁移
alembic upgrade head

# 回滚一个版本
alembic downgrade -1

# 生成迁移（注意：JSONB/UUID 类型在 SQLite 上 autogenerate 会失败，建议手写迁移）
alembic revision --autogenerate -m "描述"
```

## 3. 功能详解

### 3.1 题目体检（免费）

上传问卷题目（Word 或纯文本），系统通过 LLM 完成：
- **R1 题型识别**：单选/多选/李克特量表等
- **R2 维度归属推断**：每题归入对应维度，标注【明确归属】vs【存疑待确认】
- **R3 反向题标记**：识别反向计分题

输出题目×维度归属表，用户可修改存疑项。

**示例模板**：新建项目页提供「工作满意度」「学习动机」两个示例模板，点击一键填充项目名称与题目文本，可在此基础上修改，降低使用门槛（宪法第 15 条本科毕设生简化原则）。

### 3.2 假设解析（付费）

用户用一句话描述研究假设，例如"工作满意度正向影响职业发展"，系统解析为：
- 主效应路径（预测变量 → 结果变量）
- 强度档位：弱 / 中 / 强 ≈ r 0.2 / 0.4 / 0.6

### 假设路径可视化

解析完成后，系统会以卡片形式展示每条假设路径：
- **方向箭头**：→ 表示正向影响，← 表示负向影响
- **强度 Badge**：弱 / 中 / 强三档（不暴露原始 r 值）
- **删除按钮**：可移除不需要的假设路径

### 相关矩阵编辑

系统会展示完整的相关矩阵，并标注每个单元格的来源：
- **用户假设**（蓝色 Badge）：来自一句话解析的假设路径
- **系统补全**：系统自动补全的维度间相关

**点击单元格可循环切换强度档位**：弱（0.2）→ 中（0.4）→ 强（0.6）→ 弱（0.2）...
点击后单元格值立即更新，source 标记为「用户假设」。

> 矩阵编辑会自动持久化（debounce 800ms 保存），刷新页面后保留用户编辑的值。

### 3.3 数据生成（付费）

- 用户指定样本量 + 主效应方向
- 系统按"次要维度间相关默认 r≈0.3"补全完整相关矩阵
- **透明展示**：可视化矩阵，标注【你的假设】vs【系统补全】，可编辑
- 约束反向生成，**α 达标率目标 ≥ 70%**（实测 α=0.7955）

### 3.4 统计分析（付费）

标准统计套餐：
- 描述性统计
- 信效度：Cronbach's α、KMO、Bartlett 球形检验、探索性因子
- 相关分析
- 差异检验：t 检验 / ANOVA

**统计判定分档标准**（来源：公开统计学规范，详见 [后端架构设计文档 9.2 节](./后端架构设计文档.md)）：

| 指标 | 优秀 | 良好 | 可接受 | 不达标 |
|---|---|---|---|---|
| Cronbach's α | ≥0.90 | 0.80–0.89 | 0.70–0.79 | <0.70 |
| KMO | ≥0.90 | 0.80–0.89 | 0.50–0.79 | <0.50 |
| Bartlett p | <0.001 | <0.01 | <0.05 | ≥0.05 |
| 因子载荷 | ≥0.70 | 0.60–0.69 | 0.40–0.59 | <0.40 |
| 累计方差解释率 | ≥70% | 60–69% | 50–59% | <50% |

> KMO 通过阈值已对齐学术标准（0.5 可接受），报告输出会同时给出分档标签（如「良好」）与论文标准措辞（如「信度可接受」）。

### 3.5 诊断结论 R4（付费）

不达标时，DeepSeek-R1 推理模型给出：
- 具体问题（维度、指标、指标值、阈值）
- 原因分析
- 修改建议

用户可调参重试（C 路线）或按建议改题（B 路线）。

### 3.6 报告生成（付费）

首次进入报告页时，如果尚未生成报告，系统会显示「尚未生成报告」卡片。点击「生成报告」按钮：
- 后端运行标准统计套餐（信效度分析）
- 执行 R4 智能诊断（规则匹配 + LLM 补充）
- 计算差异检验结果（基于假设路径的决策树）
- 预计耗时 10-30 秒

生成完成后，报告内容自动展示，包含：
- 总体信效度概览（α、达标维度数）
- 各维度信效度表格（含分档等级）
- 假设检验（差异分析）结果
- R4 智能诊断结论
- 样本量信息

### 3.7 报告导出（付费）

- **Word**：完整报告（统计结果 + 诊断结论 + 免责声明），页眉带 simulated 水印
- **Excel**：信效度结果 + 诊断问题分 sheet

### 3.8 模拟数据导出（付费）

独立导出模拟数据集（Excel）：
- Sheet1 模拟数据（原始数据行）
- Sheet2 元数据（`SIMULATED DATA` 水印 + 生成时间 + 项目 ID + 样本量 + 免责声明）

可用于预研分析、方法验证、教学演示。

## 5. API 调用示例

以下示例展示完整链路，使用 `dev-token`（开发态）。生产环境替换为真实用户 token。

### 5.1 创建项目

```bash
curl -X POST http://localhost:8000/api/v1/projects/ \
  -H "Authorization: Bearer dev-token" \
  -H "Content-Type: application/json" \
  -d '{"name":"我的毕业论文预演"}'
```

响应中获取 `project_id`。

### 5.2 题目体检（免费）

```bash
curl -X POST "http://localhost:8000/api/v1/questionnaire/inspect?project_id={project_id}" \
  -H "Authorization: Bearer dev-token" \
  -H "Content-Type: application/json" \
  -d '{"text":"1. 我对工作感到满意。（1-5 非常不同意-非常同意）\n2. 职业发展前景好。"}'
```

> 说明：需要配置 `DEEPSEEK_API_KEY`，否则 LLM 解析失败。

### 5.3 创建假设（付费）

```bash
curl -X POST http://localhost:8000/api/v1/simulation/hypothesis/{project_id} \
  -H "Authorization: Bearer dev-token" \
  -H "Content-Type: application/json" \
  -d '{"raw_text":"工作满意度正向影响职业发展"}'
```

响应中获取 `hypothesis_id`。

### 5.4 数据生成（付费）

```bash
curl -X POST http://localhost:8000/api/v1/simulation/generate \
  -H "Authorization: Bearer dev-token" \
  -H "Content-Type: application/json" \
  -d '{"hypothesis_id":"{hypothesis_id}","sample_size":200}'
```

### 5.5 生成报告（付费）

```bash
curl -X POST http://localhost:8000/api/v1/report/analyze/{project_id} \
  -H "Authorization: Bearer dev-token"
```

响应中获取 `report_id`。

### 5.6 导出报告（付费）

```bash
curl -X POST http://localhost:8000/api/v1/report/export/{report_id} \
  -H "Authorization: Bearer dev-token" \
  -H "Content-Type: application/json" \
  -d '{"format":"word"}' \
  -o report.docx
```

### 5.7 导出模拟数据（付费）

```bash
curl -X POST http://localhost:8000/api/v1/simulation/export-data/{project_id} \
  -H "Authorization: Bearer dev-token" \
  -o dataset.xlsx
```

## 7. FAQ

**Q1：没有 DeepSeek API Key 能用吗？**
A：项目列表、已有数据查看、导出等功能可用；但题目体检、假设解析、诊断结论等 LLM 功能不可用。请在 `server/.env` 配置 `DEEPSEEK_API_KEY`。

**Q2：开发态如何模拟付费用户？**
A：请求头带 `Authorization: Bearer dev-token`，系统自动视为 subscription 付费用户。临时测试免费层拦截可将 users 表 dev 用户 plan 改为 `free`。

**Q3：SQLite 报 "database is locked"？**
A：已配置 WAL 模式 + busy_timeout。若仍出现，检查是否有脚本长时间占用连接，验证脚本查 DB 前先 `conn.close()` 重连。

**Q4：α 达标率达不到 70%？**
A：数据生成采用约束反向生成，实测 α=0.7955。若不达标可调整假设强度档位或样本量重试。

**Q5：前端页面看不到真实数据？**
A：全链路 BFF 路由已打通（项目/体检/假设/模拟/报告），页面均接入 TanStack Query hooks 展示真实数据。若仍看不到数据，请检查：①后端服务是否运行（`http://localhost:8000/health`）；②BFF 路由是否正常转发（查看浏览器 Network 面板）；③报告页若显示「尚未生成报告」属正常状态，需先完成体检→预演→分析流程。

**Q6：UUID 查询不匹配？**
A：`sqlalchemy.Uuid` 在 SQLite 上用 `.hex`（32 位无连字符）存储。外部脚本插入 UUID 必须用 `uuid.uuid4().hex`，不能用 `str(uuid)`。

**Q7：假设强度的弱/中/强对应多少相关系数？**
A：名义值对齐 Cohen 国际标准（r 0.1/0.3/0.5），数据生成时用离散化补偿值（r 0.2/0.4/0.6）补偿李克特量表整数化的相关性衰减。对外展示用 Cohen 名义值，避免「偏大档位美化数据」的合规嫌疑。详见 [后端架构设计文档 9.5 节](./后端架构设计文档.md)。
