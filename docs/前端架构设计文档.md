# 前端骨架文档

> 本文档为前端开发的基线，覆盖技术方案、目录边界、设计 token、页面规划。
> 风格基调：**SaaS 现代极简 + cream 暖调复古老钱质感**。
> 关联文档：[立项文档](./立项文档.md) / [Agent宪法](./Agent宪法.md)

## 一、技术方案确认

### 1.1 技术栈锁定

| 类别 | 选型 | 版本 |
|------|------|------|
| 框架 | Next.js 14（App Router） | 14.2.x |
| 语言 | TypeScript（strict 模式） | 5.x |
| UI 组件库 | **shadcn/ui**（Radix UI + Tailwind，源码进项目） | 最新 |
| 样式 | Tailwind CSS + CSS variables（design tokens） | 3.4.x |
| 服务端状态 | TanStack Query v5 | 5.x |
| 客户端状态 | Zustand | 4.x |
| 表单 | react-hook-form + zod | - |
| 图标 | lucide-react | - |
| 字体 | next/font（Fraunces + Noto Sans SC + Noto Serif SC） | - |

### 1.2 为什么选 shadcn/ui（关键决策）

- **源码进项目，完全可控**：不是 npm 依赖，组件代码复制进 `components/ui/`，可任意改样式
- **CSS variables 做主题**：正好落地 cream design token，改 token 即改全局风格
- **Radix 底层无障碍扎实**：弹窗/下拉/对话框交互质量高，不重复造轮子
- **不锁定主题**：不像 Ant Design / MUI 那样要 fight 默认样式
- **Tailwind 原生**：和项目样式方案统一，不引入 emotion/styled-components

### 1.3 不选什么、为什么

| 不选 | 理由 |
|------|------|
| Ant Design / Arco / Naive | 主题强、定制重，与 cream 风格冲突；偏 Vue 生态 |
| MUI | emotion 样式方案重，与 Tailwind 混用别扭 |
| Chakra UI | 运行时样式性能成本，主题不如 shadcn 灵活 |
| Redux | Zustand 够用，Redux 样板重 |
| styled-components / emotion | Tailwind 已够用，混用增加复杂度 |
| axios | 用原生 fetch + TanStack Query，少一个依赖 |

### 2.1 目录总览

```
web/
├── app/                          # App Router 路由层（只放路由和页面组装）
│   ├── (marketing)/              # 营销页组（无需登录）
│   │   ├── page.tsx              # 首页 landing

│   │   ├── layout.tsx            # 应用 shell（顶栏 + 侧栏）

│   │   │   ├── page.tsx          # 项目列表
│   │   │   ├── new/page.tsx      # 新建项目（上传题目）

│   │   └── ...

│   └── globals.css               # 全局样式 + design token 变量

│   ├── questionnaire/            # 业务域：题目体检

│   └── report/                   # 业务域：报告
│
├── lib/
│   ├── api/                      # API 客户端（按业务域分文件）
│   ├── hooks/                    # TanStack Query hooks（按业务域分）
```

### 2.2 模块边界规则

| 目录 | 放什么 | 不放什么 |
|------|--------|---------|
| `app/` | 路由、页面组装、layout | 业务逻辑、复杂组件实现 |
| `components/ui/` | shadcn/ui 源码组件 | 业务定制逻辑（只调样式，通过 token） |
| `components/{业务域}/` | 业务组件实现 | 路由、API 调用（调 hooks） |
| `components/layout/` | AppShell、Header、Sidebar、StepNav | 业务内容 |
| `components/common/` | PageHeader、EmptyState、Watermark 等跨域通用 | 单业务域专用 |
| `lib/api/` | fetch 封装、按域分文件 | UI 逻辑 |
| `lib/hooks/` | TanStack Query hooks（useQuestionnaire 等） | 直接 fetch（走 api/） |
| `lib/stores/` | Zustand 流程状态（如 simulationStore） | 服务端状态（走 hooks） |
| `types/` | 跨模块共享类型 | 单组件内部类型 |

### 2.3 业务域划分（对应后端服务）

| 业务域 | 前端目录 | 对应后端 |
|--------|---------|---------|
| 项目管理 | `app/(app)/projects/` + `components/project/`（暂归 common） | API Routes |
| 题目体检 | `components/questionnaire/` | API Routes → Python inspector |
| 数据生成 | `components/simulation/` | API Routes → Python generator |
| 报告产出 | `components/report/` | API Routes → Python stats + reporter |
| 数据导出 | `components/export/` | API Routes → Python exporter |

## 三、页面结构规划

### 3.1 页面清单

| 页面 | 路由 | 用途 | 所属组 |
|------|------|------|--------|
| 首页 | `/` | 营销 landing，讲预演价值 | marketing |
| 定价 | `/pricing` | 三层定价展示 | marketing |
| 关于页 | `/about` | 产品定位、产品原则、FAQ、免责声明 | marketing |
| 登录 | `/login` | 微信扫码登录 | 独立 |
| 项目列表 | `/projects` | 我的项目 | app |
| 新建项目 | `/projects/new` | 上传题目文档 | app |
| 项目工作台 | `/projects/[id]` | 体检结果 + 维度归属表 | app |
| 数据生成 | `/projects/[id]/simulate` | 假设输入 + 矩阵可视化 | app |
| 报告 | `/projects/[id]/report` | 统计结果 + 诊断 + 导出 | app |
| 数据导出 | `/projects/[id]/export` | 模拟数据集导出（Excel/CSV） | app |
| 设置 | `/settings` | 账号、订单 | app |

**页面行为补充：**

- 项目列表页 `/projects`：
  - 搜索框按项目名称实时过滤（useMemo）
  - Select 状态筛选（全部/待体检/已体检/已预演/已出报告）
  - 按更新时间降序排序
  - 删除按钮 hover 显示（opacity:0 → group-hover:opacity:100）+ Dialog 确认对话框
  - 状态徽章颜色映射：draft→secondary(灰) / inspected→default(主色) / simulated→warning(橙) / analyzed→success(绿)
- 新建项目页 `/projects/new`：
  - 示例模板卡片（宪法第 15 条简化原则），点击一键填充项目名称 + 题目文本
  - QuestionUploader 为受控组件（value/onChange），支持模板填充与手动编辑
- 工作台 `/projects/[id]`：
  - ProjectStatusCard 展示状态徽章 + 阶段说明 + 创建/更新时间 + 下一步快捷入口
  - 状态→下一步映射：inspected→去数据预演 / simulated→去查看报告 / analyzed→查看报告
- 报告页 `/projects/[id]/report`：
  - GET 报告 404 时显示「尚未生成报告」卡片 + 「生成报告」按钮
  - 点击生成按钮触发 POST /api/report/{id}/analyze
  - analyze 进行中显示 LoadingState
- 数据生成页 `/projects/[id]/simulate`：
  - 假设路径可视化卡片（方向箭头 + 强度 Badge + 删除按钮）
  - 相关矩阵点击编辑（循环切换弱/中/强，0.2→0.4→0.6），debounce 800ms 自动保存
  - 矩阵 source 标注（用户假设 vs 系统补全）

### 3.2 核心用户流程

```
首页（营销）→ 登录 → 项目列表 → 新建项目（上传题目）
  → 工作台（体检结果，免费层止步于此）
  → 付费解锁
  → 数据生成（假设 + 矩阵）
  → 报告（统计 + 诊断 + 导出 Word/Excel）
  → 数据导出（模拟数据集 Excel/CSV）
```

### 3.3 布局规范

| 区域 | 布局 |
|------|------|
| marketing 组 | 单列居中，max-w-5xl，大量留白 |
| app 组 | 左侧栏（项目导航）+ 主内容区（max-w-4xl 居中） |
| 工作台 / 生成 / 报告 | 顶部 StepNav（体检→生成→报告）+ 卡片分步内容 |

## 五、设计 Token（cream 暖调）

> **铁律：所有颜色、字号、间距、圆角、阴影、边框必须从 token 取，禁止硬编码。**
> token 定义在 `styles/tokens.css`（CSS variables），被 `globals.css` 引入，`tailwind.config.ts` 引用。

### 5.1 色彩 token

#### 背景层（cream 奶油暖白）

| Token | 值 | 用途 |
|-------|-----|------|
| `--bg-base` | `#FAF7F2` | 页面主背景 |
| `--bg-surface` | `#F5F0E8` | 卡片 / 次背景 |
| `--bg-elevated` | `#FFFFFF` | 弹窗 / 浮层（略提亮） |
| `--bg-muted` | `#EBE3D5` | 表格条纹 / 分隔区 |

|-------|-----|------|

| `--ink-500` | `#7A7066` | 辅助文字（说明） |

### 5.7 字体 token

| Token | 字体 | 用途 |
|-------|------|------|
| `--font-display` | `Fraunces, "Noto Serif SC", serif` | 大标题 / 营销（复古衬线） |

### 5.8 动效 token

| Token | 值 | 用途 |
|-------|-----|------|
| `--ease-out` | `cubic-bezier(0.16, 1, 0.3, 1)` | 默认缓动 |

> 动效克制，仅在交互反馈和步骤切换时使用，不做装饰性动效（契合 SaaS 极简）。

## 六、Tailwind 配置对接

`tailwind.config.ts` 引用 token，使 Tailwind 类直接走 token：

| Tailwind 维度 | 来源 |
|---------------|------|
| `colors` | 引用 `--bg-* / --ink-* / --accent-* / --success / --warning / --error / --chart-*` |
| `fontSize` | 引用 `--text-*` |
| `spacing` | 引用 `--space-*` |
| `borderRadius` | 引用 `--radius-*` |
| `boxShadow` | 引用 `--shadow-*` |
| `borderColor` | 引用 `--border-color` |
| `fontFamily` | 引用 `--font-*` |

shadcn/ui 组件默认通过 `bg-background / foreground / primary / muted / border` 等语义 token 取色，在 `tokens.css` 里把语义 token 映射到 cream 色值即可，组件源码无需改。

## 七、开发约定

### 7.1 取值铁律

- **禁止硬编码**：任何颜色、字号、间距、圆角、阴影、边框必须从 token 取，PR review 必查
- **例外**：一次性装饰值（如某图标的特定偏移）可硬编码，但需注释说明

### 7.2 组件分层铁律

- `components/ui/` 只调样式（通过 token），不加业务逻辑
- 业务组件不直接 fetch，走 `lib/hooks/`
- 页面（`app/`）只组装，不写复杂逻辑

### 7.3 命名约定

| 类型 | 约定 | 示例 |
|------|------|------|
| 组件文件 | PascalCase | `QuestionTable.tsx` |
| hook | camelCase，use 前缀 | `useQuestionnaire` |
| store | camelCase，Store 后缀 | `simulationStore` |
| 类型 | PascalCase | `QuestionStructure` |
| token | kebab-case | `--accent-brick` |
| 路由组 | 括号小写 | `(app)` / `(marketing)` |

### 7.4 响应式断点

| 断点 | 宽度 | 用途 |
|------|------|------|
| `sm` | 640px | 手机横屏 |
| `md` | 768px | 平板 |
| `lg` | 1024px | 桌面（主目标） |
| `xl` | 1280px | 大屏 |

> 主目标桌面端（本科毕设生在 PC 写论文），移动端做基础适配，不做精细优化。

## 九、BFF 路由层设计

> 前端通过 Next.js API Routes 作为 BFF（Backend for Frontend）层，负责字段映射 + 认证注入。

### 9.1 BFF 路由清单

| 路由 | 方法 | 后端端点 | 职责 |
|------|------|---------|------|
| /api/projects | GET | GET /api/v1/projects/ | 项目列表，snake→camelCase |
| /api/projects | POST | POST /api/v1/projects/ | 创建项目 |
| /api/projects/[id] | GET | GET /api/v1/projects/{id} | 项目详情 |
| /api/projects/[id] | PATCH | PATCH /api/v1/projects/{id} | 更新项目 |
| /api/projects/[id] | DELETE | DELETE /api/v1/projects/{id} | 删除项目 |
| /api/questionnaire/[id] | POST | POST /api/v1/questionnaire/parse/{id} | 题目体检 |
| /api/simulation/[id] | GET | GET /api/v1/simulation/{id} | 获取假设+路径+矩阵（重建） |
| /api/simulation/[id]/hypothesis | POST | POST /api/v1/simulation/hypothesis/{id} | 假设解析（LLM） |
| /api/simulation/[id]/generate | POST | POST /api/v1/simulation/generate | 数据生成（组合调用：先创建假设，再生成，最后 GET 重建矩阵） |
| /api/simulation/[id]/export-data | GET | GET /api/v1/simulation/export-data/{id} | 导出模拟数据 |
| /api/report/[id] | GET | GET /api/v1/report/{id} | 获取报告 |
| /api/report/[id]/analyze | POST | POST /api/v1/report/analyze/{id} | 生成报告（统计+诊断） |
| /api/report/[id]/export | POST | POST /api/v1/report/export/{id} | 导出报告（Word/Excel） |

### 9.2 BFF 设计模式

- **字段映射**：后端 snake_case + Decimal 字符串 → 前端 camelCase + number，用 `toNumber()` 兜底
- **认证注入**：开发态用 dev-token，生产态从用户 session 取 token
- **组合调用**：simulation generate 需组合调用（先创建假设获取 hypothesis_id，再调 generate，最后 GET 重建矩阵）
- **错误透传**：后端错误码透传到前端，保持 `{code, message, details}` 结构

### 9.4 P9 页面优化要点

- **P9-1 工作台详情页**：ProjectStatusCard 替代 PageHeader actions，状态徽章颜色映射统一（draft→灰/inspected→主色/simulated→橙/analyzed→绿），下一步快捷入口按状态动态显示
- **P9-2 项目列表页**：搜索（useMemo 实时过滤）+ 状态筛选（Select）+ 更新时间降序排序 + 删除（hover 显示 + Dialog 确认）。后端 ProjectListResponse 补充 updated_at 字段，BFF 列表 GET 补充 updatedAt 映射
- **P9-3 新项目页**：QuestionUploader 从非受控改为受控组件（value/onChange），支持模板一键填充。模板按钮 onClick 同时设置 name + rawText，降低本科毕设生使用门槛
- **P10 统计可视化增强**：报告页新增纯 SVG 图表（零依赖，不引入 recharts）。ReliabilityChart 信效度柱状图（α+KMO 分组柱+阈值虚线，达标绿色/不达标红色）；EffectSizeChart 效应量水平柱状图（显著砖红/不显著灰色，标注 p 值+效应量值）。图表放在表格前提供视觉概览，SVG fill 直接用 CSS variables 映射 design tokens
