# 数据分析智能体 立项文档

> 本文档为 6 轮产品深聊结论的沉淀，作为后续技术开发的基线。任何范围变更应以本文档为准。

## 一、项目立项说明

- **项目名**：数据分析智能体（暂定）
- **一句话定位**：面向本科毕设生的**问卷研究预演工具**
- **价值主张**：提前模拟数据方向及趋势，避免问卷"发出去石沉大海 / 收回来信效度不达标 / 相关性不显著"的白做工
- **合规底线**：
  - 定位为研究预演工具（A 路线），**非**数据生成工具（B 路线，学术造假帮凶）
  - 平台**拒绝**"题目不改强行造达标数据"的诉求
  - 全程 simulated 水印 + 免责声明，数据集标注"simulated"
  - 对外口径一律为「研究预演 / 问卷可行性验证」

## 二、目标用户

- **核心人群**：本科毕设生
- **痛点链**：问卷回收难 → 信效度不达标 → 相关性不显著 → 推翻重来
- **人群画像**：
  - 90% 使用成熟量表（李克特 5/7 级）
  - 统计需求标准化：信效度 + 相关 + 多元回归 + 差异检验（t/ANOVA）
  - 付费意愿低但量大，定价走"低价高频"
  - 季节性强：3-5 月答辩季为流量峰值，9-11 月开题季次峰值

## 三、核心功能与边界

### 第一版 MVP（必须有）

1. **题目输入**：Word / 纯文本粘贴上传
2. **题目体检（R1~R3）**：题型识别、维度归属推断、反向题标记 → 输出归属表（含【明确归属】vs【存疑待确认】标注，用户可改）
3. **假设解析（A 体验）**：用户写一句话假设 → LLM 解析为主效应路径 + 强度档位。强度名义值对齐 Cohen 国际标准（弱/中/强 = r 0.1/0.3/0.5），数据生成时采用离散化补偿值（r 0.2/0.4/0.6）以补偿李克特量表整数化的相关性衰减，避免「偏大档位美化数据」的合规嫌疑。详见 [后端架构设计文档 9.5 节](./后端架构设计文档.md)
4. **数据生成（A 体验 + C 底层）**：
   - 用户指定份数 + 主效应方向（选档）
   - 系统按"次要维度间相关默认 r≈0.3"补全完整相关矩阵
   - **透明展示**：可视化矩阵，标注【你的假设】vs【系统补全】，可编辑
5. **统计分析**：描述性统计 + 信效度（α/KMO/Bartlett/探索性因子）+ 相关分析 + 差异检验（t/ANOVA）
6. **诊断结论（R4）**：不达标时给原因 + 修改建议（B 路线为主，C 路线辅助调参重试）
7. **报告导出**：Word + Excel，含 simulated 水印
8. **导出模拟数据**：独立数据导出服务，支持 Excel/CSV 格式，含 simulated 水印元数据，可用于预研分析、方法验证、教学演示
9. **三层定价落地**：免费体检（结构 only）+ 9.9 单次报告 + 19.9/月订阅

### 第二版扩展

- 问卷星链接解析（需鉴权与反爬评估）
- PDF 报告导出 / PPT 模式
- 题目修改建议自动改写（R5）
- 进阶统计（回归、中介调节——往硕博人群扩）
- 报告文字润色（R6）
- 教程站（SEO 引流引擎）

### 暂不做

- 多人协作、团队账号
- 问卷发放功能（不碰真实数据收集，避免与问卷星正面竞争）
- AI 自动写问卷题目（偏离预演主线）

## 四、产品流程与用户路径

```
上传题目文档（Word/纯文本）
      ↓
[题目体检引擎 R1~R3]  ← 免费层
      ↓
输出题目×维度归属表（含存疑标注，可改）
      ↓
   付费解锁  ← 9.9 单次 / 19.9 月度
      ↓
[假设解析 A 体验] 用户写一句话假设
      ↓
[数据生成 A+C] 指定份数+主效应，透明展示矩阵可编辑
      ↓
[统计分析引擎] 描述/信效度/相关/差异
      ↓
[诊断结论 R4] 如不达标：原因+修改建议（B）；用户可调参重试（C）
      ↓
[报告渲染] Word + Excel（含 simulated 水印）
      ↓
[数据导出] Excel/CSV 模拟数据集（含 simulated 元数据）
      ↓
预演结论页：可行性结论 + 真实发放 checklist
```

## 五、核心业务对象

```
Project（预演项目）
  └─ Questionnaire（题目结构：题号/题型/维度/反向标记/选项映射）
       └─ SimulationConfig（生成参数：份数/期望相关矩阵/人口学分布）
            └─ Dataset（模拟数据集，含 simulated 水印元数据）
                 └─ Report（报告 + 诊断结论）
```

## 六、技术路线

- **前端**：Next.js 14（App Router）+ TypeScript strict + Tailwind + TanStack Query
- **后端**：Next.js API Routes（业务编排）+ Python FastAPI（统计/数据生成重计算）
- **LLM 双模型架构**：
  - R1~R3（理解/推断/解析）：DeepSeek-V3（快、便宜、中文够用）
  - R4（硬伤诊断推理）：DeepSeek-R1（推理模型，思维链适合因果诊断）
- **统计引擎**：scipy + statsmodels + factor_analyzer + pingouin
- **数据生成**：约束反向生成（给定相关矩阵 + 边际分布，反推数据；**用户已 PoC 验证可行**）
- **报告导出**：python-docx（Word）+ openpyxl（Excel）
- **存储**：Postgres（项目/用户）+ 对象存储（生成文件）
- **部署**：Web SaaS（公开站）

| 层级 | 价格 | 内容 | 成本 |
|------|------|------|------|
| 免费 | 0 | 题目体检 + 维度归属（**不含 R4 诊断**） | ≈0.05 元/次 |
| 单次 | 9.9 | 完整预演报告（早鸟套餐含 1 次免费重跑） | ≈0.3 元/次 |
| 订阅 | 19.9/月 | 不限次（给开题季重度用户） | 按使用量 |

**早鸟套餐策略**：不降价，多给 1 次免费修改重跑（边际成本 0.3 元，感知价值高，不伤锚点价）

## 八、项目爆点与差异点

1. **范式转换**：事后验证 → 事前预演（品类级机会，竞品都在分析"已有数据"，本产品帮用户判断"还没发的问卷值不值得发"）
2. **双模型架构**：DeepSeek-R1 推理诊断作为对外卖点（"国内首个用推理模型做问卷诊断的预演工具"）
3. **透明展示**：LLM 推断 + 可视化矩阵 + 可编辑，避免黑箱失控
4. **合规护城河**：坚守预演定位，拒绝灰色诉求，建立正向口碑

## 九、验收标准（V1~V9，已锁定）

| # | 验收点 | 关键指标 | 优先级 |
|---|--------|---------|--------|
| V1 | 上传 Word/纯文本题目，解析出题干列表 | - | **P0** |
| V2 | LLM 识别题型、维度归属、反向题，输出归属表（含存疑标注） | - | **P0** |
| V3 | 用户写一句话假设，系统解析为主效应路径+强度档位 | - | **P0** |
| V4 | 按份数+期望趋势生成数据 | **α达标率 ≥ 70%** | **P0** |
| V5 | 跑标准统计套餐：描述+信效度+相关+差异检验 | - | **P0** |
| V6 | R4 诊断结论：不达标时给原因+修改建议 | - | **P1** |
| V7 | 导出 Word + Excel，含 simulated 水印 | - | **P1** |
| V8 | 导出模拟数据（Excel/CSV），含 simulated 元数据 | - | **P2** |
| V9 | 免费层只给 V1+V2，V3~V8 付费解锁 | - | **P1** |

**优先级说明**：
- **P0**：MVP 核心链路，缺一不可（上传 → 体检 → 假设 → 生成 → 统计）
- **P1**：MVP 完整体验，上线前必须完成（诊断 + 导出 + 付费）
- **P2**：可延后，不影响核心流程（独立数据导出）

## 十、成本结构

- 单次预演 LLM 成本 ≈ 0.2 元（V3×4 次 + R1×1 次）
- 算力（数据生成 + 统计计算）≈ 0.1 元
- **单次总成本 ≈ 0.3 元，9.9 定价毛利率 97%**

## 十一、后期规划

- **教程站**（产品跑通、有付费用户后启动）：作为增长放大器，非闭环本身。Next.js 同框架 `/tutorials` 扩展。LLM 生成初稿 + 人工校对，首批 30~50 篇覆盖核心 SEO 关键词
- **硕博人群扩展**：进阶统计（SEM/CFA/中介调节）
- **B 站视频引流**：10~20 个短视频教程

## 十二、非功能需求

| 维度 | 要求 | 备注 |
|------|------|------|
| **并发量** | 初期 ≤ 50 并发 | MVP 阶段，答辩季峰值预估 |
| **响应时间** | 体检 ≤ 10s，生成 ≤ 30s，报告 ≤ 15s | LLM 调用为主要耗时 |
| **数据保留** | 项目数据保留 1 年，模拟数据集保留 30 天 | 用户可手动删除 |
| **可用性** | 99%（非关键业务） | 允许计划内维护窗口 |
| **安全** | HTTPS、密码哈希、敏感数据加密 | 基础安全要求 |
| **监控** | 错误日志 + 基础性能指标 | 初期不要求 APM |
